EdgeX 서비스 흐름도

분산형 엣지 컴퓨팅 네트워크에서 AI 서비스가 작동하는 방식

기존 클라우드 vs EdgeX 흐름 비교

기존 클라우드 기반 AI 서비스

1

사용자 요청

사용자가 AI 서비스(이미지 생성, 텍스트 분석 등)를 요청

2

중앙 데이터센터 전송

요청이 멀리 떨어진 중앙 데이터센터로 전송됨 (높은 지연 시간)

3

대규모 서버에서 처리

대규모 전용 서버에서 AI 연산 처리 (높은 비용)

4

결과 반환

처리된 결과가 다시 먼 거리를 거쳐 사용자에게 전달

주요 문제점

  • • 높은 지연 시간 (100-500ms)
  • • 고비용 인프라 (전용 서버)
  • • 중앙 집중형 장애 위험
  • • 확장성 제한

EdgeX 엣지 컴퓨팅 기반 AI 서비스

1

사용자 요청

사용자가 AI 서비스(이미지 생성, 텍스트 분석 등)를 요청

2

지역 기반 라우팅

가장 가까운 지역의 EdgeX 노드로 요청 자동 라우팅 (낮은 지연)

3

분산 네트워크에서 처리

가까운 PC방의 유휴 GPU에서 분산 처리 (저비용, 고효율)

4

신속한 결과 반환

물리적으로 가까운 거리에서 신속하게 결과 전달

주요 장점

  • • 낮은 지연 시간 (30-70ms)
  • • 30-50% 낮은 비용
  • • 분산형 장애 대응 능력
  • • 유연한 확장성

EdgeX AI 서비스 처리 워크플로우 상세

1. 요청 수신

API 게이트웨이 처리

사용자의 서비스 요청이 EdgeX API 게이트웨이에 도달하면, 요청 유효성 검사 및 사용자 인증이 진행됩니다. 이 과정에서 요청 형식, 접근 권한, 토큰 잔액 등을 확인합니다.

처리 단계: 사용자 인증 → 요청 검증 → 라우팅 준비
소요 시간: 5-10ms

2. 워크로드 분석

서비스 요구사항 분석

요청된 AI 서비스의 유형(이미지 생성, 텍스트 분석, 음성 처리 등)과 규모에 따라 필요한 컴퓨팅 자원량과 특성을 분석합니다. 이 단계에서 최적의 실행 계획이 수립됩니다.

처리 단계: 서비스 유형 분류 → 리소스 요구량 계산 → 우선순위 설정
소요 시간: 10-15ms

3. 노드 선택

최적 엣지 노드 할당

지능형 노드 선택 알고리즘이 사용자 위치, 현재 네트워크 상태, 가용 자원을 고려하여 최적의 PC방 노드를 선택합니다. 이때 지연 시간, 자원 가용성, 비용 등이 종합적으로 고려됩니다.

처리 단계: 지리적 근접성 분석 → 자원 가용성 확인 → 최적 노드 선택
소요 시간: 5-10ms

4. 작업 분배

워크로드 분배 및 실행

요청이 선택된 엣지 노드(PC방 XR7 디바이스)로 전송되고, 해당 노드에서 AI 작업을 실행합니다. 대규모 작업의 경우 여러 노드에 분산 처리될 수 있습니다.

처리 단계: 요청 전송 → 자원 할당 → AI 모델 로드 → 연산 실행
소요 시간: 서비스에 따라 다름 (10ms-수초)

5. 결과 처리

결과 취합 및 최적화

AI 작업 완료 후 결과를 수집하고, 필요시 후처리를 진행합니다. 분산 처리된 경우 여러 노드의 결과를 취합하여 최종 결과를 생성합니다.

처리 단계: 결과 수집 → 품질 검증 → 후처리 → 최적화
소요 시간: 5-20ms

6. 응답 전송

사용자에게 결과 반환

최종 처리된 결과를 사용자에게 전송합니다. 이 과정에서 토큰 결제가 처리되고, 서비스 이용 기록이 블록체인에 안전하게 저장됩니다.

처리 단계: 토큰 결제 처리 → 결과 전송 → 이용 기록 저장
소요 시간: 5-10ms

전체 처리 시간: 평균 30-70ms (기존 클라우드 대비 50-70% 단축)

이미지 생성 서비스

텍스트 프롬프트를 바탕으로 AI가 새로운 이미지를 생성하는 서비스

처리 흐름

  1. 1. 텍스트 프롬프트 분석 및 최적화
  2. 2. 적합한 이미지 생성 모델 선택
  3. 3. 고성능 GPU 노드에 작업 할당
  4. 4. 분산 추론 처리 (필요시)
  5. 5. 이미지 품질 개선 및 최적화
  6. 6. 결과 이미지 전송

성능 지표

  • • 평균 처리 시간: 1-3초
  • • 기존 대비 속도: 30% 향상
  • • 비용 절감: 약 40%

처리 흐름

  1. 1. 입력 텍스트 전처리 및 토큰화
  2. 2. 언어 모델 선택 및 로드
  3. 3. 분산 또는 단일 노드 처리

  5. 5. 결과 텍스트 후처리
  6. 6. 최종 결과 전송

성능 지표

  • • 평균 처리 시간: 50-300ms
  • • 기존 대비 속도: 50% 향상
  • • 비용 절감: 약 50%

개발자를 위한 워크플로우

1. EdgeX 개발자 계정 등록

개발자 포털에서 계정을 생성하고 API 키를 발급받습니다. 이 과정에서 개발자 신원 확인 및 기본 크레딧이 제공됩니다.

제공 도구:

  • 개발자 대시보드
  • API 문서
  • SDK 패키지
  • 샘플 코드

2. 서비스 개발 및 테스트

EdgeX SDK를 활용하여 AI 서비스를 개발하고, 개발자 환경에서 테스트합니다. 지원되는 다양한 AI 프레임워크와 모델을 활용할 수 있습니다.

지원 프레임워크:

  • TensorFlow
  • PyTorch
  • ONNX
  • Hugging Face

3. 서비스 최적화 및 배포

개발된 AI 서비스를 EdgeX 엣지 환경에 최적화하고, 배포 도구를 통해 네트워크에 배포합니다. 이 과정에서 자동 스케일링 및 분산 처리 설정이 가능합니다.

최적화 도구:

  • 모델 경량화
  • 양자화
  • 분산 처리 설정
  • 자동 스케일링

4. 서비스 모니터링 및 관리

배포된 서비스의 성능, 사용량, 수익을 실시간으로 모니터링하고 관리합니다. 분석 도구를 통해 서비스 개선 포인트를 파악할 수 있습니다.

관리 기능:

  • 실시간 대시보드
  • 사용량 분석
  • 성능 모니터링
  • 수익 분석

5. 수익화 및 확장

마켓플레이스를 통해 서비스를 판매하거나, API 형태로 제공하여 수익을 창출합니다. 사용자 피드백을 바탕으로 서비스를 지속적으로 개선하고 확장할 수 있습니다.

수익 모델:

  • 사용량 기반
  • 구독형
  • 프리미엄 기능
  • API 라이센싱

EdgeX 서비스 이용 방법

1. 계정 생성 및 토큰 획득

EdgeX 플랫폼에 사용자 계정을 생성하고, 서비스 이용을 위한 EdgeX 토큰(EXT)을 획득합니다.

  • • 웹사이트 또는 앱에서 회원가입
  • • 토큰 구매 또는 신규 사용자 보너스
  • • 지갑 설정 및 토큰 충전

2. 서비스 탐색 및 선택

마켓플레이스에서 다양한 AI 서비스를 검색하고 자신에게 필요한 서비스를 선택합니다.

  • • 카테고리별 서비스 탐색
  • • 사용자 리뷰 및 평점 확인
  • • 무료 체험판 테스트

3. 서비스 이용 및 결제

선택한 서비스를 이용하고, 사용량에 따라 자동으로 토큰이 지불됩니다.

  • • 웹 인터페이스 또는 API 연동
  • • 자동 토큰 결제 시스템
  • • 사용량 및 비용 실시간 모니터링

이용 가능한 접근 방식

웹 인터페이스

브라우저를 통해 직관적인 웹 인터페이스에서 AI 서비스를 쉽게 이용할 수 있습니다. 별도의 기술 지식 없이도 누구나 쉽게 사용 가능합니다.

모바일 앱

Android 및 iOS 앱을 통해 모바일 환경에서도 동일한 AI 서비스를 이용할 수 있습니다. 언제 어디서나 편리하게 접근 가능합니다.

API 통합

개발자는 REST API를 통해 자신의 애플리케이션에 EdgeX AI 기능을 통합할 수 있습니다. 다양한 언어의 SDK와 상세 문서를 제공합니다.
